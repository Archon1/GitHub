Lambda Expressions:

A lambda expression allows creation of an anonymous class instance that implements a functional interface. The hitBall() method, in the AnonymousClassBall{} class, meets that definition. A functional interface must contain one abstract method, and may contain other concrete methods.

Lambda functions are a quick way to create anonymous classes that implement functional interfaces. Instead of defining a formal method declaration which includes a return-type, method-name, parameter-types, and method-body-code, you just define teh parameter types and method-body-commands. The Java compiler infers the rest, based on the context of the Lambda expression. Example: () -> { System.out.println("You hit the ball!");} // this Lambda-Expression implements the interface whose abstract method takes no parameters. "->" = 'Arrow Operator'

For the anonymous class above, the coding would look like this:

public class LambdaBall { public static void main(string[] args) { Ball b = () -> {System.out.println("You hit the Ball!");}; // notice 2 semicolons: 1=end of embedded lambda-command,2=end of main() command b.hitBall(); }

interface Ball // interface for LambdaBall { void hitBall(); } }

STATIC FIELDS & METHODS:

Static fields and methods are asociated with an entire Class, not a specific Object which is an instance of the Class. It's not necessary to crate an 'instance of the class' (object) in order to access a Static Variable or Static Method.

Uses/reasons for Static fields and methods: 1. To provide constants or other values that aren't tied to a spacific class instance (example: a tax rate applicable to all taxable items) 2. To keep a count of how many instances of a class have been created or are active 3. To maintain a reference value (e.g. invoice numbers, or serial numbers which are incremented or generated according a set of rules). 4. To provide alternative ways to crete class instances. Example: Java NumberFormat class has getCurrencyInstance() and getNumberInstance that return Object instances to format numeric values in specific ways. Note: the NumberFormat example also describes a 'Singleton Class' and Singleton Pattern. 5. To provide a set of utility functions to a Class that aren't associated with a specific object. Examples include many types of data validation.

Static variables or fields can be declared private outside of any class methods. BUT... ... Because static variables and methods are available to an entire class (all instances or objects of the Class Type), static variables can't be declared in private methods.

INTERFACES:

An interface is a set of constants and method declarationss that define the form of a Class (What the class can do), but with no implementations (nothing stating How the class accomplishes its functions)

Variables declared in an Interface are (implicitly) static, public, and final; meaning, they are constants. Functions prototyped in an Interface are (implcitly) Public and Abstract; they have no implementation

The default Modifier for an interface provides access to code in the same Package [package-level] A 'Public' Modifier for an interface allows for access outside the Package (access from other Packages)

A class can implement many interfaces -- but can only extend (inherit-from) one other class.

Interfaces can extend other interfaces.

Abstract Classes v. Interfaces:

They are Similar, but an Abstract Class is inherited, an Interface is implemented. Java interface are invoked using the keyword ''implements''; A Java abstract class is extended (inherited) using keyword ''extends''.

A Class is abstract if any of its methods is declared as Abstract A class extending an Abstract Class must implement ALL of the abstract methods of the parent class, or must be itself an Abstract class. An abstract class can have 'concrete' methods (instance methods that implement 'default' behavior) Interface methods are all public and abstract. No Interface methods can be implemented in the Interface. Interface variables are all public, static, and final. Variables declared in a Java interface are final (by default). An abstract class may contain non-final variables. Members of a Java interface are public by default. A Java abstract class can have the usual flavors of class members .. private, protected, etc.. Like a Java Class, a Java interface can extend another Java interface, A Java class can implement multiple interfaces but it can extend only one abstract class An Abstract class has constructors, even if they are not declared or implemented in the Abstract Class. Inerfaces have no constructors.
